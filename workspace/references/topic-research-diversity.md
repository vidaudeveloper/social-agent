# 选题搜索为何总是「大差不差」

## 常见原因

1. **关键词太泛**  
   Step 1 若反复搜「TikTok 小店」「选品趋势」「GMV Max」，WebSearch 返回的 SEO 洗稿高度同质。

2. **只走单一信源**  
   只用 `web_search`、不用小红书搜索 / 知乎热榜 / RSS，结果会集中在少数门户的同一批话题。

3. **用户画像未细化**  
   `user-profile.md` 里行业、受众、竞品账号为空时，Agent 会套用跨境带货通用模板选题。

4. **时间窗口固定**  
   总搜「最近 24–48h 热点」但不换角度（政策 / 案例 / 工具 / 踩坑 / 数据），标题结构会雷同。

5. **模型与训练偏置**  
   大模型对「TK 封号」「选品公式」「ROAS」等高频词有强先验，不加强约束会反复推荐类似角度。

6. **缺少「已写过」记忆**  
   未对照 `content/文章/` 历史 slug，容易重复同一话题变体。

## Agent 在 Step 1 必须做到

1. **读 `user-profile.md`**：行业、受众、竞品、已启用平台
2. **多信源交叉**（至少 2 个）：`web_search` + 小红书 `search-feeds` / 知乎热榜 / `blogwatcher` RSS
3. **关键词多样化**：每条候选附**不同切入角**（数据解读 / 单点案例 / 工具对比 / 反常识 / 政策变化）
4. **显式去重**：列出 `content/文章/` 近 30 天标题，候选不得与历史标题高度相似
5. **标注来源 URL**，不要只复述摘要
6. **5–8 条候选中至少 2 条「非爆款套路」**（长尾问题、细分人群、区域市场）

## 搜索词模板（替换泛词）

| 避免 | 可换成 |
|------|--------|
| TikTok 选品趋势 | TikTok Shop 类目 X 差评关键词 / 某国退货率 |
| 封号原因 | 某违规类型 2026 判例 / 申诉通过率 |
| GMV Max 教程 | 某预算档位 ROAS 分布 / 类目冷启动失败案例 |

## 用户可主动说

- 「换一批，不要 TK 小店/封号/选品」
- 「从 user-profile 的竞品账号里找选题」
- 「对照 content 目录，不要重复上周题目」
